What is Empower Desktop Pool by App Volumes

Enpower Desktop Pool by App Volumes tool provides an easy way to associate existing Horizon View desktop pools with App Volumes. App Stacks managed by App Volumes can be attached to users/desktops from a desktop pool perspective.

Why Using It?

By using this tool to attach App Stacks to desktop pool, base OS will be separated from applications, and application upgrade will be less distruptive in the future: no recreation of virtual desktops, no recompose (for Linked-Clone pool), and fast.

Setup Guide

Preconditions
  1. Existing Horizon View environment
  2. App Volumes manager is installed, and App Stacks have been prepared. (Official web site for App Volumes)
  3. App Volumes agent is installed in desktops of target pools.

Working with manual pool

There is no further configuration for manual pool, except the preconditions. Use this tool attach App Stacks to a selected desktop pool, that's all.

Working with automated pool

Automated pool (linked clone, or not) will have desktop dynamically created. To make sure newly created desktops have App Stacks automatically attached as configured for the owner desktop pool (configured in this tool), an additional change is required for the desktop pool master VM template:

  1. Download this autoAttach.exe (right click the link, select Save As...)
  2. configure it to be auto started when Windows starts.

The application will communicate with this Migrate to App Volumes tool which is installed on View Connection server. The script will attach App Stacks which are configured for its owner desktop pool. in future, desktop pool related App Stack attach/detatch can be operated in this Migrate to App Volumes tool, so as to avoid changing of desktop pool master VM template for this purpose.

For more information, contact VMware EUC Technical Enablement team.
